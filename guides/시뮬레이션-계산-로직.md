# 은퇴 시뮬레이션 계산 로직 명세서 v2

> **목적**: 이 문서는 `은퇴계산기.tsx`의 `calculateSimulation()` 함수를 완벽히 재현하기 위한 기술 명세입니다.
> 이 문서만 보고 동일한 계산 결과를 내는 코드를 작성할 수 있어야 합니다.
> **최종 업데이트**: 2026-03-22

---

## 전체 아키텍처

### 파일 구조
| 파일 | 역할 |
|------|------|
| `은퇴계산기.tsx` | 입력폼 UI + `calculateSimulation()` 계산 엔진 + 결과 테이블 |
| `src/ActionPlan.tsx` | 우측 패널: 성공/실패 판정 표시 + 전략 제안 + 전략 반영 기능 |
| `src/App.tsx` | 전체 레이아웃, 페이지 라우팅, 시뮬레이션 결과 상태 관리 |

### 데이터 흐름
```
[사용자 입력 (inputs)]
  → [calculateSimulation()]
  → [SimulationRow[] 결과]
  → [결과 테이블 렌더링] + [ActionPlan 패널에 전달]
  → [ActionPlan에서 전략 제안]
  → [전략 반영 시 inputs 수정 → 자동 재계산]
```

## 계산 흐름 요약

```
[0단계] 초기 잔액 결정 (적립식/직접입력 모드 분기, 복수 계좌 합산)
   ↓
[1단계] 연도별 루프 시작 (retirementStartAge ~ simulationEndAge)
   ↓
[2단계] 고정 수입 계산 (국민연금, 주택연금, 생명보험연금)
   ↓
[3단계] 변동 인출 계산 (소진모드 ON/OFF 분기)
   ↓
[4단계] 건강보험료 재계산
   ↓
[5단계] 부채 상환 + 비정기 지출
   ↓
[6단계] 총계 계산 + 추가차감(500만) + 부족분 보전
   ↓
[7단계] 연말 수익률 적용 (인출 후 잔액에 복리)
   ↓
[8단계] 시뮬레이션 완료 후 성공/실패 판정 + 원인 분석
```

## 0단계: 초기 잔액 결정

### 적립식 복리 계산 공식 (FV)
ISA, overseas, savings, personalPension에 공통 적용:
```
n = retirementStartAge - currentAge
r = returnRate
cb = currentBalance (현재 잔액)
yc = yearlyContribution (연간 납입액)

if n <= 0 → 잔액 = cb
if r === 0 → 잔액 = cb + yc × n
else → 잔액 = cb × (1+r)^n + yc × ((1+r)^n - 1) / r
```

### 추가자산 잔액 결정
```javascript
for (const asset of inputs.additionalAssets) {
  const bal = (assetInputMode === 'accumulate' && ['isa','overseas','savings'].includes(asset.type))
    ? FV(asset.currentBalance, asset.yearlyContribution, asset.returnRate, yearsToRetire)
    : asset.balance;

  if (asset.type === 'isa') resolvedISA = bal;
  if (asset.type === 'overseas') resolvedOverseas = bal;
  if (asset.type === 'savings') resolvedSavings = bal;
}
```

### 퇴직연금 합산
```javascript
if (inputs.retirementPensions.length > 0) {
  totalRetirementBalance = SUM(rp.balance)
  weightedRetirementRate = SUM(rp.balance × (rp.type === 'db' ? 0 : rp.returnRate)) / totalRetirementBalance
} else {
  // 하위 호환
  totalRetirementBalance = inputs.retirementPensionBalance
  weightedRetirementRate = inputs.retirementPensionType === 'db' ? 0 : inputs.retirementPensionReturnRate
}
```

### 개인연금 합산
```javascript
if (inputs.personalPensions.length > 0) {
  for (const pp of inputs.personalPensions) {
    ppBal = (pensionInputMode === 'accumulate') ? FV(...) : pp.balance;
    totalPersonalPensionBalance += ppBal;
    weightedPersonalRate += ppBal × pp.returnRate;
  }
  weightedPersonalRate /= totalPersonalPensionBalance;
} else {
  totalPersonalPensionBalance = inputs.totalPension;
  weightedPersonalRate = inputs.pensionReturnRate;
}
```

### 합산 연금 잔액 (시뮬레이션 시작값)
```
pensionBalance = totalPersonalPensionBalance + totalRetirementBalance
pensionReturnRate = 가중평균(개인연금Rate, 퇴직연금Rate)
```

| 변수 | 값 |
|------|-----|
| isaBalance | resolvedISA |
| pensionBalance | 개인연금합산 + 퇴직연금합산 |
| overseasBalance | resolvedOverseas |
| savingsBalance | resolvedSavings |
| debtBalance | inputs.totalDebt |

## 2단계: 고정 수입 계산

### 생활비 (지출)
```
baseMonthly = currentAge < 75 ? monthlyLivingCostBefore75 : monthlyLivingCostAfter75
livingCost = baseMonthly × 12 × (1 + inflationRate/100)^i
```

### 주택연금
```
homePension = currentAge >= homePensionStartAge ? homePensionMonthly × 12 : 0
```

### 생명보험연금
```
lifeInsurancePension = currentAge >= lifeInsurancePensionStartAge ? lifeInsurancePensionYearly : 0
```

### 국민연금 (세후)
```javascript
if (currentAge >= nationalPensionStartAge) {
  // 1. 조기/연기 조정계수
  if (개시나이 < 65) adjustmentFactor = 1 - min(5, 65-개시나이) × 0.06
  else if (개시나이 > 65) adjustmentFactor = 1 + min(5, 개시나이-65) × 0.072
  else adjustmentFactor = 1

  // 2. 물가상승 반영
  yearsFromStart = currentAge - nationalPensionStartAge
  adjustedPension = nationalPensionYearly × adjustmentFactor × (1 + inflationRate/100)^yearsFromStart

  // 3. 나이별 세율 적용
  nationalPension = adjustedPension × (1 - ageTaxRate)
}
```

## 3단계: 변동 인출 계산

### 공통: 건보료 예측 (인출 전)
```
propertyScore = max(0, homeValue - 1억) × 0.0000005
estimatedPensionIncome = min(pensionWithdrawalAmount, pensionBalance) × 0.945
estimatedHealthInsurance = (estimatedPensionIncome × 0.002/1000 + propertyScore) × 218.8 × 12
```

### 공통: 필요액
```
fixedIncome = nationalPension + homePension + lifeInsurancePension
remainingNeeded = livingCost + estimatedHealthInsurance - fixedIncome
```

### PMT 공식 (소진모드에서 사용)
```
factor = (1+r)^n
PMT = balance × (r × factor) / (factor - 1)

if r === 0 → PMT = balance / n
if n === 1 → PMT = balance
PMT = min(PMT, balance)  // 잔액 초과 방지
```

---

### 소진 모드 ON (`usePensionDepletion === true`)

모든 자산을 시뮬레이션 종료까지 PMT로 균등 소진.

```
remainingYears = simulationEndAge - currentAge + 1
```

**순서:**
1. **개인연금 소진**: `pensionRemainingYears = simulationEndAge - max(currentAge, pensionStartAge) + 1`, PMT(pensionBalance, pensionReturnRate, pensionRemainingYears)
2. **ISA 소진**: PMT(isaBalance, isaReturnRate, remainingYears)
3. **해외배당** (자동): `overseasBalance × 0.06 × (1 - 0.154)`
4. **해외주식 소진**: PMT(overseasBalance, max(0, overseasReturnRate - 0.06), remainingYears)

### 연금 세후 계산 (소진/일반 공통)
```
withinLimit = min(pensionPreTax, 15,000,000)
afterTaxWithin = withinLimit × (1 - ageTaxRate)

excessAmount = max(0, pensionPreTax - 15,000,000)
afterTaxExcess = excessAmount × (1 - pensionExcessTaxRate)

pensionAfterTax = afterTaxWithin + afterTaxExcess
```

---

### 소진 모드 OFF (`usePensionDepletion === false`)

필요한 만큼만 인출. 우선순위: 해외배당 → 개인연금 → ISA → 해외주식

1. **해외배당** (자동): `overseasBalance × 0.06 × (1-0.154)`, remainingNeeded에서 차감
2. **개인연금**: `actualWithdrawal = min(remainingNeeded/0.945, pensionWithdrawalAmount, pensionBalance)`
3. **ISA**: `isaWithdrawal = min(max(0, remainingNeeded), isaBalance)`
4. **해외주식 매도**: `overseasStockSale = min(max(0, remainingNeeded), overseasBalance)`

## 4단계: 건강보험료 재계산

개인연금 인출 확정 후 정확한 건보료 산출:
```
propertyScore = max(0, homeValue - 1억) × 0.0000005
incomeScore = pensionAfterTax × 0.002 / 1000
healthInsurance = (incomeScore + propertyScore) × 218.8 × 12
```

> **2026년 기준 보수적 계산**: 개인연금 세후 인출액만 소득 반영.
> 국민연금, ISA, 해외소득은 건보료 계산에서 제외.
> 향후 공적연금 건보료 반영 확대 가능성 있음.

## 5단계: 부채 상환 + 비정기 지출

if (debtBalance > 0 && currentAge <= debtEndAge)
  debtRepayment = min(monthlyDebtRepayment × 12, debtBalance)

irregularExpense = SUM(expense.amount) where expense.age === currentAge

## 6단계: 총계 계산 + 부족분 보전

### 총계
```
totalIncome = pensionAfterTax + nationalPension + homePension + lifeInsurancePension
            + isaWithdrawal + overseasDividend + overseasStockSale
totalExpense = livingCost + healthInsurance + debtRepayment + irregularExpense
```

### 추가 차감 (안전 마진)
```
additionalDeduction = currentAge >= nationalPensionStartAge ? 5,000,000 : 0
yearlySurplus = totalIncome - totalExpense - additionalDeduction
```
> 국민연금 수령 시점부터 연 500만원 추가 차감 (의료비/비정기 지출 버퍼)

### 부족분 보전 (yearlySurplus < 0일 때)
인출 우선순위: **예적금 → ISA → 해외주식**
```
각 단계:
  use = min(부족분, 해당잔액)
  해당잔액 -= use
  totalIncome += use
  yearlySurplus += use
```

---

## 7단계: 연말 수익률 적용

인출 후 남은 잔액에 복리 적용:
```
isaBalance      *= (1 + isaReturnRate)
overseasBalance *= (1 + max(0, overseasReturnRate - 0.06))  ← 배당률 차감, 최소 0%
pensionBalance  *= (1 + pensionReturnRate)
savingsBalance  *= (1 + savingsReturnRate)
```

> **해외주식 수익률 < 6% 경고**: 잔액 성장 0%, 배당만 수령. UI에서 경고 표시.

| 원인 | 조건 |
|------|------|
| 생활비 과다 | livingCost > totalExpense × 50% |
| 자산소득 부족 | 연금+해외소득 < livingCost × 70% |

| 자산 고갈 위험 | 총자산 < 초기자산 × 30% |

## ActionPlan 전략 제안 로직

### 실패 시: 적자 총액 역산 방식
```
totalDeficit = |min(0, totalSurplus)|
coverTarget = totalDeficit × 1.3  (30% 안전 마진)
```

전략별 커버 비율:
| 전략 | 비율 | 계산 방법 | 변경 필드 |
|------|------|---------|----------|
| 은퇴 시점 연기 | 40% | `ceil(budget / avgLivingCost)` 년 | retirementStartAge +N |
| 생활비 절감 | 25% | `ceil(budget / adjSimYears / 12)` 원 | monthlyLivingCostBefore75, After75 |
| 추가 저축 | 20% | `ceil(budget / 1000만)` 원 | savingsAmount +N |
| 수익률 상향 | 15% | +2%p 고정 | isaReturnRate, overseasReturnRate, savingsReturnRate |
| 국민연금 연기 | 추가 | +5년 (최대 70세) | nationalPensionStartAge |

### 성공 시: 보수적 최적화
- 수익률 +1%p, 생활비 -50만, 국민연금 +1년 (고정값)

### 전략 반영 시 제한값
| 필드 | 제한 |
|------|------|
| nationalPensionStartAge | 최대 70세 |
| monthlyLivingCostBefore75/After75 | 최소 100만원 |
| 모든 수치 | 최소 0 |

### 전략 반영 후 자동 재계산
`onApplyStrategies(changes, true)` → `autoRecalculate` 플래그 → inputs 수정 후 자동 `calculateSimulation()` 실행

### 전략 반영 표시 (PLAN 배지)
- `appliedFields` state로 반영된 필드 추적
- 반영된 입력 필드 옆에 오렌지 `PLAN` 배지 표시
- 초기화/프리셋 선택 시 배지 초기화

## UI 동작 규칙

### 계산하기 버튼
- `canCalculate` 조건: 생활비 > 0 또는 자산/연금 1개 이상 존재
- 미충족 시 disabled (회색, 반투명)
- 상단 (프리셋 옆) + 하단 2곳에 동일 `runSimulation()` 함수 사용

### 우측 패널 (ActionPlan)
- 계산하기 실행 시 열림 (`showActionPanel = true`)
- 초기화 시 닫힘 (빈 results 전달 → `showActionPanel = false`)
- `transition: width 0.25s ease` 애니메이션 적용

### 프리셋 (나와 비슷한 상황 보기)
- 커스텀 드롭다운 (네이티브 select 아님)
- 미선택 시: 파란 배경 + shimmer 애니메이션
- 선택 시: 일반 스타일로 전환
- 선택 시 `assetInputMode`, `pensionInputMode` → 'direct'로 리셋

### 최근계산 버튼
- localStorage에 데이터 없으면 disabled
- 우측 끝 배치 (`marginLeft: auto`)

### 입력값 변경 감지
- 계산 후 입력값 변경 시 `inputsDirty = true`
- "다시 계산하기" 텍스트로 변경

## 핵심 주의사항 (재구현 시 반드시 확인)

1. **연금 잔액은 퇴직연금 + 개인연금 합산**으로 관리 (`pensionBalance` 하나)
2. **해외주식 수익률은 배당 6% 차감 후 적용**, `max(0, rate-0.06)` (마이너스 성장 방지)
3. **PMT 공식은 소진모드에서만 사용** (일반모드는 필요분만 인출)
4. **부족분 보전 우선순위**: 예적금 → ISA → 해외주식
5. **건보료 소득 기준**: 개인연금 세후 인출액만 반영
6. **추가 차감 500만원**: 국민연금 수령 나이부터 매년 적용
7. **수익률 적용 시점**: 인출 후 잔액에 연말 복리
8. **적립식 모드**: FV 공식으로 은퇴 시점 잔액 산출 후 시뮬레이션 투입
9. **전략 반영 후 자동 재계산**: `autoRecalculate` 플래그로 setTimeout 체인 실행
10. **전략은 적자 총액 × 1.3 기반 역산**: 모든 전략 적용 시 성공 보장
